SPI (MAX7219)
  • On this project you will use the SPI (Serial Peripheral Interface) component
  • PSoC will communicate with a MAX7219, that is a 7-segment display controller
  • In SPI there is a master and multiple slaves
  • When the SPI master wants to communicate with a slave, the first thing to do is to activate the SS signal (Slave Select or Chip Select) connected to the slave
  • To send data to the slave, the master uses the MOSI signal (Master-Output Slave-Input) and the clock signal (SCLK)
  • IMPORTANT : The clock (SCLK) always is generated by the SPI master
  • In order to receive data from the slave, the master must send any date trough the MOSI signal, together with the clock (SCLK). The slave will send the data back to the master by using the MISO signal (Master-Input Slave-Output)
  • That is the only confusing point on the SPI bus. In order to receive data, the master has to send any data to the slave. This data is dismissed by the slave, and it is normally called DUMMY
  • Here PSoC will communicate with only one SPI slave (MAX7219) and will not use the MISO signal, because the master will only write the slave registers
  • The master sends information to the slave, but the slave never sends information back to the master
  • Change the jumper J9 so that the pins 2 and 3 are connected. So the kit will run with 5 volts. That is needed because the MAX7219 needs 5 volts
  • This is the SPI configuration used by the SPI slave MAX7219:
Signal Description
SS (Slave Select) The SS signal is active low
SCLK (Clock) In idle state, SCLK is low and the data are captured on the rising edge of SCLK
MOSI The SPI master will send always 2 bytes. The first is the address of the register to write and the second the data.
  • The bits D8 to D11 contain the address of the register to write.
    The bits D0 to D7 the data
  • There are 14 registers:
  • The "Decode Mode" register (address 0x09) controls the BCD code.If it is 1, the BCD code is active and you can say to the MAX7219 to show a character (i.e. "1") instead of having to activate all LEDs that form a digit one to one
  • In the following table you see how the BCD mode can be activated for no digit, for the digit 0, for the digits 0 to 3, or for all
  • On this project you will use all digits coded in BCD, that is why a 0xFF value is written into the register 0x09
  • In the following table, you can see in red the character to show and in green the value you need to write
  • If you wanted to show the character "1" on the digit 2, you would have to write on the register "Digit 2" (address 0x03) the value 0x01.
    If you want also to show the decimal point the value to write would be 0x81
  • The intensity register (address 0x0A) controls the light intensity of the LED diodes
  • In the following table you can see that the intensity can be configured from the minimum level (value 0x00) to the maximum level (value 0x0F).
    You will use the value 0x02
  • The "Scan-Limit" (address 0x0B) controls which digits will be activated
  • You will use all digits, so you need to write the value 0x07
  • The "Shutdown" register (address 0x0C) configures the display in normal mode
  • For normal mode, a value of 0x01 has to be written
  • The registers configuration will be as follows:
Register Address Value
Decode-Mode 0x09 0xFF
Intensity 0x0A 0x02
Scan-Limit 0x0B 0xFF
Shutdown 0x0C 0x01
Steps
  • Create a new project
  • Based on the CY8CKIT-042 kit
  • With an empty schematic
  • And rename it as workspace WS022
  • Add the "SPI Master" component to the schematic
  • Edit its properties
  • Rename it as "SPIM" and configure "Mode" as "CPHA=0, CPOL=0"
  • The reception and transmission buffer size is 4
  • Add a digital input
  • Edit its properties
  • Rename it as "Pin_MISO"
  • Connect "Pin_MISO" to the input "miso" of "SPIM"
  • Add a digital input
  • Edit its properties
  • Rename it as "Pin_MOSI"
  • Connect it to the output "mosi" of "SPIM"
  • Copy "Pin_MOSI" and paste it. Edit the properties of the new digital output
  • Rename it as "Pin_SCLK"
  • Connect it to the output "sclk" of "SPIM"
  • Add another digital output, pasting "Pin_MOSI".
    Edit its properties and rename it as "Pin_CS".
    Uncheck "Hw connection" (it will be controlled by program)
  • Add a clock
  • Edit its properties
  • Rename it as "SPI_CLOCK" and set 1MHz as the frequency.
    The SCLK frequency will be the half = 500 HKz
